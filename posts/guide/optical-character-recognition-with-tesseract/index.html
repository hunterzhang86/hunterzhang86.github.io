使用 Tesseract 进行光学字符识别

 2023.4.9  2023.4.8  手册  2357  5 分钟  

一、概述

随着人工智能和机器学习技术的进步，我们需要工具来识别图像中的文本。

在本教程中，我们将探索 Tesseract，一种光学字符识别 (OCR) 引擎，并提供一些图像到文本处理的示例。

  1. 正方体

Tesseract 是惠普开发的开源 OCR 引擎，识别 100 多种语言，并支持表意和从右到左的语言. 另外，我们可以训练 Tesseract 识别其他语言。

它包含两个用于图像处理的 OCR 引擎——一个 LSTM（长短期记忆）OCR 引擎和一个通过识别字符模式工作的传统 OCR 引擎。

OCR 引擎使用 Leptonica 库 打开图像并支持**各种输出格式，如纯文本、hOCR（用于 OCR 的 HTML）、PDF 和 TSV。* *

  1. 正方体命令行

4.1。跑

我们可以使用 Tesseract 命令行工具从图像中提取文本。

例如，让我们对我们的网站进行快照： [](/ wp-content/uploads/2020/03/Screen-Shot-2020-02-28-at-6.29.53-PM.png)

然后，我们将运行 tesseract 命令来读取 baeldung.png 快照并将文本写入 output.txt 文件：

1

tesseract baeldung.png 输出

output.txt 文件将如下所示：

一个带有 Spring 的 REST 学习 Spring（新！）
构建产品的规范参考
使用 Spring 分级 API。
从没有经验到实际构建东西。
是的
Java 每周评论

我们可以观察到 Tesseract 并没有处理图像的全部内容。因为输出的准确性取决于各种参数，例如图像质量、语言、页面分割、训练数据和用于图像处理的引擎。

4.2。语言支持

默认情况下，OCR 引擎在处理图像时使用英语。但是，我们可以使用 -l 参数声明语言：

让我们看另一个多语言文本的例子： [](/ wp-content/uploads/2020/03/Screen-Shot-2020-03-08-at-10.43.12-AM.png)

首先，让我们使用默认的英语语言处理图像：

1

tesseract multiLanguageText.png 输出

输出将如下所示：

Der ,.schnelle" braune Fuchs springt

然后，让我们用葡萄牙语处理图像：

1

tesseract multiLanguageText.png 输出 -l por

因此，OCR 引擎也会检测葡萄牙语字母：

Der ,.schnelle" braune Fuchs springt

同样，我们可以声明语言的组合：

1

tesseract multiLanguageText.png 输出 -l spa+por

在这里，OCR 引擎将主要使用西班牙语和葡萄牙语进行图像处理。但是，输出可能会根据我们指定的语言顺序而有所不同。

4.3。页面分割模式

Tesseract 支持多种页面分割模式，如 OSD、自动页面分割和稀疏文本。

我们可以使用 --psm 参数来声明页面分割模式，其值为 0 到 13 用于各种模式：

1

tesseract multiLanguageText.png 输出 --psm 1

在这里，通过定义值 1，我们声明了使用 OSD 进行自动页面分割以进行图像处理。

我们来看看所有支持的页面分割模式： [](/ wp-content/uploads/2020/03/Screen-Shot-2020-03-08-at-2.28.30-PM.png)

同样，我们可以在处理图像时使用各种引擎模式，如 legacy 和 LSTM 引擎。

为此，我们可以使用值为 0 到 3 的 --oem 参数：

1

tesseract multiLanguageText.png 输出 --oem 1

4.5。苔丝数据

Tesseract 包含 LSTM OCR 引擎的两组训练数据——最佳训练的 LSTM 模型 和 [训练的 LSTM 模型的快速整数版本](https: //github.com/tesseract-ocr/tessdata_fast).

前者提供更好的准确性，后者提供更好的图像处理速度。

此外，Tesseract 提供了一个组合训练数据，支持旧版和 LSTM OCR 引擎。

如果我们使用 Legacy OCR 引擎而不提供支持的训练数据，Tesseract 将抛出错误：

1
2
3

错误：请求了 Tesseract（旧版）引擎，但 /usr/local/share/tessdata/eng.traineddata 中不存在组件！
加载语言“eng”失败
Tesseract 无法加载任何语言！

因此，我们应该下载所需的 .traineddata 文件并将它们保存在默认的 tessdata 位置或使用 --tessdata-dir 参数声明该位置：

1

tesseract multiLanguageText.png 输出 --tessdata-dir /image-processing/tessdata

4.6。输出

我们可以声明一个参数来获取所需的输出格式。

例如，要获得可搜索的 PDF 输出：

1

tesseract multiLanguageText.png 输出 pdf

这将在提供的图像上创建带有可搜索文本层（带有可识别文本）的 output.pdf 文件。

同样，对于 hOCR 输出：

1

tesseract multiLanguageText.png 输出 hocr

此外，我们可以使用 tesseract --help 和 tesseract --help-extra 命令来获取有关 tesseract 命令行用法的更多信息。

5.苔丝4J

Tess4J 是 Tesseract API 的 Java 包装器，它为各种图像格式（如 JPEG、GIF、PNG 和 BMP）提供 OCR 支持。

首先，让我们将最新的 tess4j Maven 依赖项添加到我们的 pom.xml 中：

    <artifactId>tess4j</artifactId>
    <版本>4.5.1</版本>
</依赖>

六，结论

在本文中，我们通过几个图像处理示例探索了 Tesseract OCR 引擎。

首先，我们检查了用于处理图像的 tesseract 命令行工具，以及一组参数，例如 -l 、 --psm 和 --oem。

然后，我们探索了 tess4j，这是一个将 Tesseract 集成到 Java 应用程序中的 Java 包装器。
