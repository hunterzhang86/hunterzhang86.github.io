如何使用正则表达式替换标记

 2023.1.24  2023.4.14  手册  3598  8 分钟  

一、概述

当我们需要在Java中查找或替换字符串中的值时，我们通常使用正则表达式。这些允许我们确定 部分或全部字符串匹配 模式。我们可以在 Matcher 和 String 中使用 replaceAll 方法轻松地 **对字符串中的多个标记应用相同的替换。

在本教程中，我们将探讨如何对字符串中的每个标记应用不同的替换。这将使我们更容易满足转义某些字符或替换占位符值等用例。

我们还将研究一些调整正则表达式以正确识别标记的技巧。

  1. 单独处理比赛

在我们构建我们的逐个令牌替换算法之前，我们需要了解正则表达式周围的 Java API。让我们使用捕获组和非捕获组来解决一个棘手的匹配问题。

2.1。标题案例示例

假设我们想要构建一个算法来处理字符串中的所有标题词。这些单词以一个大写字符开头，然后以小写字符结束或继续。

我们的输入可能是：

1

“前 3 个大写单词！然后是 10 个 TLA，我找到了”

根据标题词的定义，这包含匹配项：

  • 第一的
  • 首都
  • 字
  • 我
  • 成立

识别这种模式的正则表达式是：

1

“(?<=^|[^A-Za-z])([A-Z][a-z]*)(?=[^A-Za-z]|$)”

为了理解这一点，让我们把它分解成它的组成部分。我们将从中间开始：

1

[A-Z]

将识别单个大写字母。

我们允许单字符单词或单词后跟小写，所以：

1

[a-z]*

识别零个或多个小写字母。

在某些情况下，上述两个字符类足以识别我们的标记。不幸的是，在我们的示例文本中，有一个单词以多个大写字母开头。因此，我们需要表示我们找到的单个大写字母必须是第一个出现在非字母之后的。

同样，由于我们允许单个大写字母单词，我们需要表示我们找到的单个大写字母不能是多大写字母单词的第一个。

表达式 [^A-Za-z] 表示“没有字母”。我们将其中一个放在非捕获组中的表达式开头：

1

(?<=^|[^A-Za-z])

以 (?<=, 开头的非捕获组执行 look-behind 以确保匹配出现在正确的边界。 它最后的对应组对后面的字符执行相同的工作。

但是，如果单词触及字符串的开头或结尾，那么我们需要考虑这一点，这就是我们添加 ^| 的地方。到第一组以使其表示“字符串的开头或任何非字母字符”，我们在最后一个非捕获组的末尾添加了 |$ 以允许字符串的结尾是边界。

当我们使用 find 时，在非捕获组中找到的字符不会出现在匹配中。

我们应该注意，即使是像这样的简单用例也可能有很多边缘情况，所以测试我们的正则表达式很重要。为此，我们可以编写单元测试，使用我们 IDE 的内置工具，或使用 Regexr 等在线工具。

  1. 一些用例

5.1。转义特殊字符

假设我们想使用正则表达式转义字符 \ 手动引用正则表达式的每个字符，而不是 使用 quote 方法。也许我们正在引用一个字符串作为创建正则表达式以传递给另一个库或服务的一部分，因此块引用表达式是不够的。

如果我们可以表达意味着“正则表达式字符”的模式，那么使用我们的算法很容易将它们全部转义：

1
2
3
4
5
6

模式 regexCharacters = Pattern.compile("[<(\\[{\\\\^\\-=$!|\\]})?*+.>]");

assertThat(replaceTokens("一个像 [",
  正则表达式字符，
  匹配 -> "\\" + match.group()))
  .isEqualTo("一个正则表达式字符，如 \\[");

对于每个匹配，我们在 \ 字符前面加上前缀。由于 \ 是 Java 字符串中的一个特殊字符，它被另一个 \ 转义。

实际上，这个示例包含了额外的 \ 字符，因为 regexCharacters 模式中的字符类必须引用许多特殊字符。这显示了正则表达式解析器，我们使用它们来表示它们的文字，而不是作为正则表达式语法。

5.2。替换占位符

表达占位符的常用方法是使用 ${name} 之类的语法。让我们考虑一个用例，其中模板 "Hi ${name} at ${company}" 需要从名为 placeholderValues 的映射中填充：

1
2
3

Map<String, String> placeholderValues = new HashMap<>();
placeholderValues.put("name", "Bill");
placeholderValues.put("公司", "Baeldung");

我们只需要一个好的正则表达式来查找 ${...} 标记：

1

“\\$\\{(?<占位符>[A-Za-z0-9-_]+)}”

是一种选择。它必须引用 $ 和初始花括号，否则它们将被视为正则表达式语法。

此模式的核心是占位符名称的捕获组。我们使用了一个允许字母数字、破折号和下划线的字符类，它应该适合大多数用例。

但是，为了使代码更具可读性，我们将此捕获组命名为 占位符。让我们看看如何使用该命名捕获组：

1
2
3
4

assertThat(replaceTokens("Hi ${name} at ${company}",
  "\\$\\{(?<占位符>[A-Za-z0-9-_]+)}",
  匹配-> placeholderValues.get(match.group("placeholder"))))
  .isEqualTo("你好比尔在 Baeldung");

在这里我们可以看到，从 Matcher 中获取命名组的值只涉及使用 group 将名称作为输入，而不是数字。
