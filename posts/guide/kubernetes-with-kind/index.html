Kubernetes 有种

 2023.1.5  2023.4.28  手册  2652  6 分钟  

一、概述

在使用 Kubernetes 时，我们缺少一个有助于本地开发的工具——一个可以使用 Docker 容器作为节点运行本地 Kubernetes 集群的工具。

在本教程中，我们将使用 kind 探索 Kubernetes。 kind 主要是 Kubernetes 的测试工具，对于本地开发和 CI 也很方便。

  1. 设置

作为先决条件，我们应该确保 Docker 已安装在我们的系统中。安装 Docker 的一种简单方法是使用适合我们的操作系统（和处理器，如果是 macOS）的 Docker Desktop。

2.1。安装 Kubernetes 命令行

首先，让我们安装 Kubernetes 命令行，kubectl。在 macOS 上，我们可以使用 Homebrew 安装它：

1

$ 酿造安装 kubectl

我们可以使用以下命令验证安装是否成功：

1
2
3
4
5

$ kubectl 版本 --client

客户端版本：version.Info{Major:"1", Minor:"21", GitVersion:"v1.21.3",
GitCommit:"ca643a4d1f7bfe34773c74f79527be4afd95bf39", GitTreeState:"clean",
BuildDate：“2021-07-15T21:04:39Z”，GoVersion：“go1.16.6”，编译器：“gc”，平台：“darwin/amd64”}

同样，我们可以在 Windows 上使用 curl 进行下载：

1

curl -LO https://dl.k8s.io/v1.21.0/bin/windows/amd64/kubectl.exe.sha256

然后，我们应该将 kubectl 命令的二进制位置添加到我们的 PATH 变量中。

2.2。安装 种类

接下来，我们将在 macOS 上使用 Homebrew 安装 kind：

1

$ brew install 种类

要验证安装是否成功，我们可以尝试以下命令：

1
2

$种版本
kind v0.11.1 go1.15.6 darwin/amd64

但是，如果 kind version 命令不起作用，请将其位置添加到 PATH 变量中。

同样，对于 Windows 操作系统，我们可以使用 curl 下载 kind：

1
2

curl -Lo kind-windows-amd64.exe https://kind.sigs.k8s.io/dl/v0.11.1/kind-windows-amd64
移动项目 .\kind-windows-amd64.exe c:\kind\kind.exe

3.Kubernetes集群

现在，我们都准备好使用 kind 为 Kubernetes 准备本地开发环境了。

3.1。创建集群

首先，让我们使用默认配置创建一个本地 Kubernetes 集群：

1

$ kind 创建集群

默认情况下，将创建一个名为 kind 的集群。但是，我们可以使用 --name 参数为集群提供一个名称：

此外，我们可以使用 YAML 配置文件来配置集群。例如，让我们在 baeldungConfig.yaml 文件中编写一个简单的配置：

1
2
3

种类：集群
api版本：kind.x-k8s.io/v1
名称: baeldung-kind

然后，让我们使用配置文件创建集群：

1

$ kind 创建集群 --config baeldungConfig.yaml

此外，我们还可以在创建集群时提供特定版本的 Kubernetes 镜像：

1

$ kind 创建集群 --image kindest/node:v1.20.7

3.3。集群详细信息

集群准备就绪后，我们可以在 kubectl 上使用 cluster-info 命令检查详细信息：

1
2
3

$ kubectl cluster-info --context kind-baeldung-kind
Kubernetes master 运行在 https://127.0.0.1:59489
CoreDNS 在 https://127.0.0.1:59489/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy 运行

此外，我们可以使用 dump 参数和 cluster-info 命令来提取有关集群的详细信息：

1

$ kubectl cluster-info dump --context kind-baeldung-kind

3.4。删除集群

与 get 命令类似，我们可以使用 delete 命令删除特定集群：

1

$ kind 删除集群 --name baeldung-kind

4.入口控制器

4.1。配置

我们需要一个 ingress 控制器来在我们的本地环境和 Kubernetes 集群之间建立连接。

- 角色：控制平面
  kubeadmConfigPatches：
  - |
    种类：初始化配置

      kubeletExtraArgs：

  额外端口映射：

  - 容器端口：443

在这里，我们更新了我们的 baeldungConfig.yaml 文件来设置入口控制器的配置，将容器端口映射到主机端口。此外，我们通过定义 ingress-ready=true 为入口启用了节点。

然后，我们必须使用修改后的配置重新创建集群：

1

kind 创建集群 --config baeldungConfig.yaml

4.2。部署

然后，我们将部署 Kubernetes 支持的 ingress NGINX 控制器 作为反向代理和负载均衡器：

1

kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml

此外，我们还可以使用 AWS 和 [GCE 负载均衡器](https://git.k8s.io/ingress -gce/README.md#readme) 控制器。

5.2。部署

现在我们已经准备好所有配置并且我们的集群与入口 NGINX 控制器集成，让我们部署我们的服务：

1

$ kubectl apply -f baeldung-service.yaml

1

注意：如果我们遇到任何与 validate.nginx.ingress.kubernetes.io webhook 相关的错误，我们应该删除 ValidationWebhookConfiguration：

1
2

$ kubectl delete -A ValidatingWebhookConfiguration ingress-nginx-admission
验证webhookconfiguration.admissionregistration.k8s.io“ingress-nginx-admission”已删除

然后，再次部署服务。

六，结论

在本文中，我们使用 kind 探索了 Kubernetes。

首先，我们做了一个设置，包括安装 Kubernetes 命令行 kubectl 和 kind 。然后，我们通过 kind 的一些功能来创建/更新 Kubernetes 本地集群。

最后，我们集成了入口控制器并在 Kubernetes 集群上部署了一个私有可访问的服务。
