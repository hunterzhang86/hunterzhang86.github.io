Jinq with Spring 简介

1。介绍

Jinq 为使用 Java 查询数据库提供了一种直观方便的方法。在本教程中，我们将探讨如何配置 Spring 项目以使用 Jinq 以及通过简单示例说明其一些特性。

3。了解金q

Jinq 通过公开一个内部基于**Java Stream API 的流畅 API 帮助我们编写更简单、更易读的数据库查询api/java.base/java/util/stream/package-summary.html)**

让我们看一个按型号过滤汽车的示例：

1
2
3

jinqDataProvider.streamAll(entityManager, Car.class)
  .where(c -> c.getModel().equals(model))
  .toList();

Jinq 以一种有效的方式将上述代码片段转换为 SQL 查询，因此本示例中的最终查询将是：

1

select c.* from car c where c.model=?

由于我们不使用纯文本来编写查询，而是使用类型安全的 API，因此这种方法不太容易出错。

此外，Jinq 旨在通过使用通用、易于阅读的表达式来加快开发速度。

然而，它在我们可以使用的类型和操作的数量方面有一些限制，我们将在下面看到。

3.1。限制

Jinq 仅支持 JPA 中的基本类型和 SQL 函数的具体列表。 它通过将所有对象和方法映射到 JPA 数据类型和 SQL 函数来将 lambda 操作转换为原生 SQL 查询。

因此，我们不能指望该工具翻译每个自定义类型或一个类型的所有方法。

3.2。支持的数据类型

让我们看看支持的数据类型和方法：

  • String -- equals() , compareTo() 仅限方法
  • 原始数据类型——算术运算
  • Enums 和自定义类——仅支持 == 和 != 操作
  • java.util.Collection -- contains()
  • Date API -- equals() , before() , after() 方法

注意：如果我们想自定义从 Java 对象到数据库对象的转换，我们需要在 Jinq 中注册 AttributeConverter 的具体实现。

5。查询指南

Jinq 提供了许多直观的选项来自定义最终的 SQL 查询，包括 select、where、 joins 等。 请注意，这些限制与我们上面已经介绍的限制相同。

5.1。在哪里

where 子句允许将多个过滤器应用于数据集合。

在下一个示例中，我们要按型号和描述过滤汽车：

选择 c.model, c.description from car c where c.model=?并定位（？，c.description）> 0

5.2。选择

如果我们只想从数据库中检索几个列/字段，我们需要使用 select 子句。

为了映射多个值，Jinq 提供了多个 Tuple 类，最多包含八个值：

  .toList()

以及翻译后的 SQL：

从汽车 c 中选择 c.model、c.year、c.engine
