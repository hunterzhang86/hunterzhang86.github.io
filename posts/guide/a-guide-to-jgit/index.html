JGit 指南

 2023.3.11  2023.4.18  手册  1979  4 分钟  

1。介绍

JGit 是 Git 版本控制系统的轻量级纯 Java 库实现——包括存储库访问例程、网络协议和核心版本控制算法。

JGit 是用 Java 编写的一个功能比较全的 Git 实现，在 Java 社区中被广泛使用。 JGit 项目在 Eclipse 保护伞下，它的主页可以在 JGit 找到。

在本教程中，我们将解释如何使用它。

2。入门

有多种方法可以将您的项目与 JGit 连接并开始编写代码。可能最简单的方法是使用 Maven——通过将以下代码片段添加到我们的 pom.xml 文件中的 <dependencies> 标记来完成集成：

1
2
3
4
5

<依赖>
    <groupId>org.eclipse.jgit</groupId>
    <artifactId>org.eclipse.jgit</artifactId>
    <version>4.6.0.201612231935-r</version>
</依赖>

请访问 Maven 中央存储库 获取最新版本的 JGit。完成此步骤后，Maven 将自动获取并使用我们需要的 JGit 库。

如果您更喜欢 OSGi 包，还有一个 p2 存储库。请访问 Eclipse JGit 以获取如何集成此库的必要信息。

3。创建存储库

JGit 有两个基本级别的 API：plumbing 和 porcelain。这些术语来自 Git 本身。 JGit分为相同的区域：

  • porcelain APIs -- 常见用户级操作的前端（类似于 Git 命令行工具）
  • plumbing API——直接与低级存储库对象交互

大多数 JGit 会话的起点是 Repository 类。我们要做的第一件事是创建一个新的 Repository 实例。

init 命令将让我们创建一个空的存储库：

1

Git git = Git.init().setDirectory("/path/to/repo").call();

这将在提供给 setDirectory() 的位置创建一个带有工作目录的存储库。

可以使用 cloneRepository 命令克隆现有存储库：

1
2
3
4

git git = Git.cloneRepository()
  .setURI("https://github.com/eclipse/jgit.git")
  .setDirectory("/path/to/repo")
  。称呼（）;

上面的代码会将 JGit 存储库克隆到名为 path/to/repo 的本地目录中。

4。 Git 对象

在 Git 对象模型中，所有对象都由 SHA-1 id 表示。在 JGit 中，这由 AnyObjectId 和 ObjectId 类表示。

Git 对象模型中有四种类型的对象：

  • blob -- 用于存储文件数据
  • tree -- 一个目录；它引用了其他 trees 和 blob
  • commit -- 指向一棵树
  • tag -- 将提交标记为特殊；通常用于标记特定版本

要从存储库中解析对象，只需传递正确的修订版本，如下面的函数：

1

ObjectId head = repository.resolve("HEAD");

4.1。 参考

Ref 是一个保存单个对象标识符的变量。对象标识符可以是任何有效的 Git 对象（blob、tree、commit、tag）。

例如，要查询对 head 的引用，您可以简单地调用：

1

Ref HEAD = repository.getRef("refs/heads/master");

4.2。 RevWalk

RevWalk 遍历提交图并按顺序生成匹配的提交：

1

RevWalk walk = new RevWalk(repository);

4.3。 修订提交

RevCommit 代表 Git 对象模型中的提交。要解析提交，请使用 RevWalk 实例：

1
2

RevWalk walk = new RevWalk(repository);
RevCommit 提交 = walk.parseCommit(objectIdOfCommit);

4.4。 修订标签

RevTag 代表 Git 对象模型中的标签。您可以使用 RevWalk 实例来解析标签：

1
2

RevWalk walk = new RevWalk(repository);
RevTag 标签 = walk.parseTag(objectIdOfTag);

4.5。 RevTree

RevTree 代表 Git 对象模型中的一棵树。 RevWalk 实例也用于解析树：

1
2

RevWalk walk = new RevWalk(repository);
RevTree 树 = walk.parseTree(objectIdOfTree);

5。瓷API

虽然 JGit 包含许多与 Git 存储库一起使用的低级代码，但它还包含一个更高级别的 API，它模仿 org.eclipse.jgit.api 包中的一些 Git porcelain 命令。

5.1。 AddCommand (git-add)

AddCommand 允许您通过以下方式将文件添加到索引中：

  • 添加文件模式()

下面是如何使用 porcelain API 将一组文件添加到索引的快速示例：

1
2
3

Git git = new Git(db);
AddCommand add = git.add();
add.addFilepattern("someDirectory").call();

5.2。 CommitCommand (git-commit)

CommitCommand 允许您执行提交，并提供以下选项：

  • setAuthor()
  • setCommitter()
  • 全部设置()

下面是一个如何使用 porcelain API 提交的快速示例：

1
2
3

Git git = new Git(db);
CommitCommand 提交 = git.commit();
commit.setMessage("初始提交").call();

5.3。 TagCommand (git-tag)

TagCommand 支持多种标记选项：

  • setName()
  • setMessage()
  • setTagger()
  • setObjectId()
  • setForceUpdate()
  • setSigned()

下面是一个使用 porcelain API 标记提交的快速示例：

1
2
3

Git git = new Git(db);
RevCommit commit = git.commit().setMessage("initial commit").call();
RevTag tag = git.tag().setName("tag").call();

5.4。 LogCommand (git-log)

LogCommand 允许您轻松浏览提交图。

  • 添加（AnyObjectId 开始）
  • addRange（AnyObjectId 以来，AnyObjectId 直到）

这是一个如何获取一些日志消息的快速示例：

1
2

Git git = new Git(db);
可迭代<RevCommit> log = git.log().call();

6。蚂蚁任务

JGit 在 org.eclipse.jgit.ant 包中还包含一些常见的 Ant 任务。

要使用这些任务：

1
2
3
4
5
6
7

<taskdef resource="org/eclipse/jgit/ant/ant-tasks.properties">
    <类路径>
        <pathelement location="path/to/org.eclipse.jgit.ant-VERSION.jar"/>
        <pathelement location="path/to/org.eclipse.jgit-VERSION.jar"/>
        <pathelement location="path/to/jsch-0.1.44-1.jar"/>
    </类路径>
</taskdef>

这将提供git-clone、git-init 和git-checkout 任务。

6.1。 git-clone

1

<git-clone uri="http://egit.eclipse.org/jgit.git" />

以下属性是必需的：

  • uri: 要克隆的 URI

以下属性是可选的：

  • dest : 克隆到的目标（默认使用基于 URI 的最后一个路径组件的人类可读目录名称）
  • bare : true /false /yes /no 表示克隆的存储库是否应该是裸露的（默认为 false）
  • branch : 克隆存储库时要签出的初始分支（默认为 HEAD）

6.2。 git-init

1

<git-init />

运行 git-init 任务不需要任何属性。

以下属性是可选的：

  • dest : 初始化 git 存储库的路径（默认为 $GIT_DIR 或当前目录）
  • bare : true /false /yes /no 表示存储库是否应该是裸露的（默认为 false）

6.3。 git-checkout

1

<git-checkout src="path/to/repo" branch="origin/newbranch" />

以下属性是必需的：

  • src：git仓库的路径
  • branch：结帐的初始分支

以下属性是可选的：

  • createbranch : true /false /yes /no 表示如果分支不存在，是否应该创建它（默认为 false）
  • force : true /false /yes /no : 如果 true /yes 并且具有给定名称的分支已经存在，将设置现有分支的起点到一个新的起点；如果 false ，则不会更改现有分支（默认为 false）
