Java 16 中的新功能

 2022.7.28  2022.10.19  手册  2873  6 分钟

一、概述

Java 16 于 2021 年 3 月 16 日发布，是基于 [Java 15](/java-15-新的）。此版本带有一些有趣的功能，例如记录和密封类。

在本文中，我们将探讨其中的一些新功能。

  1. 从代理实例调用默认方法 (JDK-8159746)

作为对 Interfaces 中默认方法的增强，随着 Java 16 的发布，已将支持添加到 java.lang.reflect.InvocationHandler [调用默认方法](https://bugs.openjdk.java.net/browse /JDK-8159746) 通过使用反射的 动态代理 的接口。

为了说明这一点，让我们看一个简单的默认方法示例：

    默认字符串你好（）{
        返回“世界”；
    }

  1. 日间支持 (JDK-8247781)

DateTimeFormatter 的新增功能 是一天中的周期符号“B”，它提供了 am/pm 的替代方案格式：

1
2
3

LocalTime 日期 = LocalTime.parse("15:25:08.690791");
DateTimeFormatter 格式化程序 = DateTimeFormatter.ofPattern("h B");
assertThat(date.format(formatter)).isEqualTo("下午3点");

我们得到的不是像“3pm”这样的输出，而是“3 in the tomorrow”的输出。我们还可以分别使用“B”、“BBBB”或“BBBBB”DateTimeFormatter 模式来表示短样式、完整样式和窄样式。

  1. 添加 Stream.toList 方法 (JDK-8180352)

目的是使用一些常用的 Stream 收集器来减少样板，例如 Collectors.toList 和 Collectors.toSet：

1
2
3

List<String> integersAsString = Arrays.asList("1", "2", "3");
List<Integer> ints = integersAsString.stream().map(Integer::parseInt).collect(Collectors.toList());
List<Integer> intsEquivalent = integersAsString.stream().map(Integer::parseInt).toList();

我们的 ints 示例以旧方式工作，但 intsEquivalent 具有相同的结果并且更简洁。

  1. 矢量 API 孵化器 (JEP-338)

Vector API 处于 Java 16 的初始孵化阶段。这个 API 的想法是提供一种最终能够执行的矢量计算方法比传统的标量计算方法更优化（支持 CPU 架构）。

让我们看看传统上如何将两个数组相乘：

1
2
3
4
5
6
7
8

int[] a = {1, 2, 3, 4};
int[] b = {5, 6, 7, 8};

var c = new int[a.length];

for (int i = 0; i < a.length; i++) {
    c[i] = a[i] * b[i];
}

对于长度为 4 的数组，此标量计算示例将在 4 个周期内执行。现在，让我们看看等效的基于向量的计算：

1
2
3
4
5
6
7

int[] a = {1, 2, 3, 4};
int[] b = {5, 6, 7, 8};

var vectorA = IntVector.fromArray(IntVector.SPECIES_128, a, 0);
var vectorB = IntVector.fromArray(IntVector.SPECIES_128, b, 0);
var vectorC = vectorA.mul(vectorB);
向量C.intoArray(c, 0);

我们在基于向量的代码中做的第一件事是使用此类 fromArray. 的静态工厂方法从我们的输入数组创建两个 IntVectors。第一个参数是向量的大小，然后是数组和偏移量（这里设置为 0）。这里最重要的是我们要达到 128 位的向量的大小。在 Java 中，每个 int 需要 4 个字节来保存。

由于我们有一个 4 ints 的输入数组， 它需要 128 位来存储。我们的单个 Vector 可以存储整个数组。

在某些架构上，编译器将能够优化字节码以将计算从 4 个周期减少到仅 1 个周期。这些优化有利于机器学习和密码学等领域。

我们应该注意到，处于孵化阶段意味着这个 Vector API 可能会随着新版本的发布而改变。

记录 是在 Java 14 中引入的。Java 16 带来了一些 增量更改。

类似地，有时开发人员会跳过创建正确的 不可变类 的必要步骤。有时我们最终会重用一个通用类，而不是为每个不同的用例定义一个专门的类。

大多数现代 IDE 提供了自动生成代码（例如 setter、getter、构造函数等）的能力，这有助于缓解这些问题并减少开发人员编写代码的开销。但是，记录提供了一种内置机制来减少样板代码并创建相同的结果。

  1. instanceof 的模式匹配 (JEP-394)

instanceof 关键字的模式匹配 自 Java 16 起已添加。

以前我们可能会这样写代码：

1
2
3
4
5
6

对象 obj = "测试";

if (obj instanceof String) {
    字符串 t = (字符串) obj;
    // 做一些逻辑...
}

这段代码不能只关注应用程序所需的逻辑，而是必须首先检查 obj 的实例，然后将对象转换为 String 并将其分配给新变量 t.

随着模式匹配的引入，我们可以重写这段代码：

1
2
3
4
5

对象 obj = "测试";

if (obj instanceof String t) {
    // 做一些逻辑
}

我们现在可以声明一个变量 -- 在这个实例中是 t -- 作为 instanceof 检查的一部分。

  1. 密封课程 (JEP-397)

密封类，首先 在 Java 15 中引入，提供了一种机制来确定允许哪些子类扩展或实现父类或接口。

8.1。例子

让我们通过定义一个接口和两个实现类来说明这一点：

1
2
3
4
5
6
7
8

公共密封接口 JungleAnimal 允许 Monkey, Snake {
}

public final class Monkey 实现 JungleAnimal {
}

公共非密封类 Snake 实现 JungleAnimal {
}

sealed 关键字与 permits 关键字一起使用，以确定允许哪些类实现此接口。在我们的示例中，这是 Monkey 和 Snake.

密封类的所有继承类必须用以下之一标记：

  • sealed -- 意味着他们必须使用 permits 关键字定义允许从它继承的类。
  • final -- 阻止任何进一步的子类
  • 非密封 - 允许任何类能够继承它。

密封类的一个显着好处是它们允许详尽的模式匹配检查，而不需要对所有未覆盖的情况进行捕获。例如，使用我们定义的类，我们可以有逻辑覆盖 JungleAnimal 的所有可能子类：

1
2
3
4
5
6
7

JungleAnimal j = // 一些 JungleAnimal 实例

if (j instanceof Monkey m) {
    // 做逻辑
} else if (j instanceof Snake s) {
    // 做逻辑
}

我们不需要 else 块，因为密封类只允许 Monkey 和 Snake 这两种可能的子类型。

8.2。 Java 16 中密封类的新添加

Java 16 中有 对密封类的一些补充。这些是 Java 16 对密封类引入的更改：

  • Java 语言将 sealed 、 non-sealed 和 permits 识别为上下文关键字（类似于 abstract 和 extends）
  • 限制创建作为密封类的子类的本地类的能力（类似于无法创建密封类的匿名类）。
  • 转换密封类和从密封类派生的类时进行更严格的检查
  1. 其他变更

从 Java 15 版本中的 JEP-383 开始，外部链接器 API 提供了一种访问主机上本机代码的灵活方式。最初，为了 C 语言的互操作性，将来可能会适应其他语言，例如 C++ 或 Fortran。此功能的目标是最终取代 Java 原生接口。

另一个重要的变化是 JDK 内部现在 默认强封装。这些从 Java 9 开始就可以访问了。但是，现在 JVM 需要参数 --illegal-access=permit。这将影响当前直接使用 JDK 内部的所有库和应用程序（尤其是在测试方面），并且直接忽略警告消息。
