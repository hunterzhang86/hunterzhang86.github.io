使用 javac 编译目录结构中的所有 Java 类

 2022.7.22  2022.10.5  手册  1985  4 分钟

一、概述

在某些特殊情况下，我们没有安装第三方构建工具，例如 Ant 或 Maven。而且，我们仍然需要编译一个包含大量包和类的项目。

在本教程中，我们将使用 javac 命令在不同的场景中完成此任务。

  1. 使用文件名

假设我们在当前目录中有两个目录： src 和 out 。 src 目录保存我们的 Java 源文件，而 out 目录将包含相应的编译类文件。

让我们从一个简单的场景开始。 src 目录包含一个名为 com/baeldung/MyClass.java 的 Java 源文件： [] (/wp-content/uploads/2022/03/2_javac-compile-all-java-source-files-in-a-directory-structure-01.png)

然后，让我们使用 javac 将 MyClass.java 文件编译到 out 目录中：

1

$ javac -d ./out/ ./src/com/baeldung/MyClass.java

在上述命令中，-d 选项指定类文件的目标目录。另外，我们应该注意 MyClass.java 文件的确切代码并不那么重要，我们只需要确保它是一个语法正确的 Java 文件。

稍微复杂一点，让我们添加另外三个 Java 文件—— YourClass.java 、 HerClass.java 和 HisClass.java ： [] (/wp-content/uploads/2022/03/javac-compile-all-java-source-files-in-a-directory-structure-02.png)

要编译上述所有四个 Java 文件，我们可以在命令行中列出它们中的每一个：

1
2
3
4
5

$ javac -d ./out/ \
./src/com/baeldung/MyClass.java \
./src/com/baeldung/YourClass.java \
./src/com/baeldung/HerClass.java \
./src/com/baeldung/HisClass.java

然后，让我们添加一个引用其他四个 Java 文件的新 Main.java 文件，例如，通过调用方法或创建对象实例：[](/wp-content/uploads/2022/03/2_javac-compile-all -java-source-files-in-a-directory-structure-03.png)

在这种情况下，我们只需要编译 Main.java 文件：

1

$ javac -sourcepath ./src/ -d ./out/ ./src/com/baeldung/Main.java

在上面的命令之后，其他四个类文件也会被编译。这是因为 javac 默认会搜索所需的类型并编译相应的源文件。如果我们不想编译所需的类型，我们可以添加 -implicit:none 选项。

-sourcepath 选项告诉 Java 编译器在哪里可以找到输入源文件。如果没有指定 -sourcepath 选项，javac 将使用用户类路径来搜索类文件和源文件。因此，我们可以将 -sourcepath 选项替换为 -classpath 或 -cp 选项：

1

$ javac -cp ./src/ -d ./out/ ./src/com/baeldung/Main.java

但是，这种方法有其局限性：javac 命令仅编译所需的类型并忽略其他源文件。例如，如果我们添加一个新的 ItsClass.java 而 Main.java 没有引用它，那么 ItsClass.java 将不会被编译： [] (/wp-content/uploads/2022/03/2_javac-compile-all-java-source-files-in-a-directory-structure-04.png)

总而言之，有两种情况适合在 javac 命令行中列出文件名：当只有几个 Java 源文件时，以及当有一个启动器类递归地引用其他类时。

  1. 使用通配符

javac 命令还支持使用通配符 (*) 来编译同一目录中的多个源文件。

例如，我们可以使用通配符来编译上面的源文件：

1

$ javac -d ./out/ ./src/com/baeldung/*.java

为了使我们的场景更加复杂，让我们添加四个子包（spring、summer、autumn 和 winter）和相应的类： [] (/wp-content/uploads/2022/03/javac-compile-all-java-source-files-in-a-directory-structure-05.png)

现在，在命令行中，我们可以使用通配符列出每个包来编译它们：

1
2
3
4
5
6

$ javac -d ./out/ \
./src/com/baeldung/*.java \
./src/com/baeldung/spring/*.java \
./src/com/baeldung/summer/*.java \
./src/com/baeldung/autumn/*.java \
./src/com/baeldung/winter/*.java

当只有几个包时，无论源文件编号如何，都适合使用这种通配符的方式。

  1. 使用参数文件

当有多个包要编译时，使用带有参数文件的 javac 命令会派上用场。 参数文件可以包含 javac 选项和源文件名。

要使用参数文件，我们需要在参数文件名之前添加 at 符号 (@) 前导字符：

1

$ javac -d ./out/ @sources.txt

但是我们怎样才能生成这样一个 @sources.txt 文件呢？这取决于我们使用的操作系统。在 Linux 或 macOS 中，我们可以使用 find 命令：

1

$ find ./src/ -type f -name "*.java" > sources.txt

在上面的命令行中，./src/ 是我们搜索的起始目录，-type f 选项只过滤常规文件，-name "*.java" 选项匹配所有文件名*.java* 扩展名。

但是，在 Windows 中，我们可以使用 dir 命令：

> 目录 src /b /s *.java > 来源.txt

在上面的命令行中，src 文件夹是我们的搜索路径，/b 开关显示目录和文件名，没有附加信息，/s 选项列出指定目录中的所有文件和所有子目录。

这种方法的缺点是每次添加新的或删除现有的 Java 源文件时，都需要重新生成 sources.txt 文件。

  1. 其他方法

除了上述常用方法外，还存在其他依赖于操作系统的方法，例如使用 globstar 或管道。

5.1。使用 Globstar

Bash 4.0 版添加了一个名为 globstar 的新通配符选项，它以不同的方式处理双通配符 (**)。启用后，Bash 将遍历多级目录；否则，Bash 将只搜索单级目录。

但是，默认情况下禁用此选项。而且，我们可以使用 shopt (sh + opt, shell options) 命令来检查 Bash 选项设置。如果我们执行不带任何参数的 shopt 命令，它将输出一长串选项及其状态（on 或 off）。

目前，我们只关心 globstar 选项：

1
2

$ shopt globstar
全球明星关闭

要启用它，我们使用带有 -s 选项的 shopt 命令：

1

$ shopt -s globstar

要禁用它，我们使用 -u 选项调用 shopt 命令：

1

$ shopt -u globstar

启用此选项后，我们可以使用双通配符**调用 javac：

1

$ javac -d ./out/ ./src/**/*.java

5.2。使用管道

从概念上讲，管道是两个进程之间的连接。而且，我们可以利用这种管道机制来连接多个命令以产生我们想要的结果。

要编译我们的 Java 源文件，我们可以组合 find 、 xargs 和 javac 命令：

1

$ find ./src/ -type f -name "*.java" | xargs javac -cp ./src/ -d ./out/

此外，find 命令支持 -exec 操作：

1

$ find ./src/ -type f -name "*.java" -exec javac -cp ./src/ -d ./out/ '{}' ';'

上面的命令行运行可能有点慢。这是因为 javac 命令将为每个匹配的文件运行。有关更多信息，我们可以使用 man find 命令阅读 -exec 选项的文档。

为了更快一点，我们可以将分号 (; ) 更改为加号 (+) 。然后，javac 命令将收集所有匹配的文件并只执行一次：

1

$ find ./src/ -type f -name "*.java" -exec javac -cp ./src/ -d ./out/ '{}' +
