Spring 表达式语言指南

 2022.7.15  2022.10.18  手册  4468  9 分钟

1。概述 --------------- Spring 表达式语言 (SpEL) 是一种强大的表达式语言，支持在运行时查询和操作对象图。我们可以将它与 XML 或基于注释的 Spring 配置一起使用。该语言有几个可用的运算符： | 类型 | 运营商 | |-------------|------------------------------------ ---------------| |算术 | +, -, *, /, %, ^, div, mod | |关系 | <, >, ==, !=, <=, >=, lt, gt, eq, ne, le, ge | |逻辑 |和，或，不，&&，||，！ | |有条件的 | ?: | |正则表达式 |比赛 | 2。运算符 ---------------- 对于这些示例，我们将使用基于注释的配置。在本文后面的部分中查找有关 XML 配置的更多详细信息。 SpEL 表达式以 # 符号开头并用大括号括起来：#{expression}。可以以类似的方式引用属性，以 $ 符号开头并用大括号括起来：${property.name}。属性占位符不能包含 SpEL 表达式，但表达式可以包含属性引用： java #{${someProperty} + 2} 在上面的示例中，假设 someProperty 的值为 2，因此结果表达式将是2 + 2，这将被评估为 4。### 2.1。算术运算符 SpEL 支持所有基本算术运算符：java @Value("#{19 + 1}") // 20 private double add; @Value("#{'String1 ' + 'string2'}") // "String1 string2" private String addString; @Value("#{20 - 1}") // 19 私有双减； @Value("#{10 * 2}") // 20 私有双乘； @Value("#{36 / 2}") // 19 私有双除法； @Value("#{36 div 2}") // 18, 同 for / operator private double divideAlphabetic; @Value("#{37 % 10}") // 7 私有双模； @Value("#{37 mod 10}") // 7，与 % operator private double moduloAlphabetic; 相同； @Value("#{2 ^ 9}") // 512 private double powerOf; @Value("#{(2 + 2) * 2 + 9}") // 17 个私有双括号； 除法和模运算有字母别名， div 代表 / 和 mod 代表 % 。 + 运算符也可用于连接字符串。 ### 2.2。关系和逻辑运算符 SpEL 还支持所有基本的关系和逻辑运算：java @Value("#{1 == 1}") // true private boolean equal; @Value("#{1 eq 1}") // 真正的私有布尔 equalAlphabetic; @Value("#{1 != 1}") // false private boolean notEqual; @Value("#{1 ne 1}") // false private boolean notEqualAlphabetic; @Value("#{1 < 1}") // false private boolean lessThan; @Value("#{1 lt 1}") // false private boolean lessThanAlphabetic; @Value("#{1 <= 1}") // 真正的私有布尔值 lessThanOrEqual; @Value("#{1 le 1}") // 真正的私有布尔值 lessThanOrEqualAlphabetic; @Value("#{1 > 1}") // false 私有布尔大于； @Value("#{1 gt 1}") // false private boolean GreaterThanAlphabetic; @Value("#{1 >= 1}") // 真正的私有 boolean GreaterThanOrEqual; @Value("#{1 ge 1}") // 真正的私有布尔值 GreaterThanOrEqualAlphabetic; 所有关系运算符也有字母别名。例如，在基于 XML 的配置中，我们不能使用包含尖括号（<、<=、>、>=）的运算符。相反，我们可以使用lt（小于）、le（小于或等于）、gt（大于）或ge（大于或等于）。 ### 2.3。逻辑运算符 SpEL 还支持所有基本逻辑运算：java @Value("#{250 > 200 && 200 < 4000}") // true private boolean and; @Value("#{250 > 200 and 200 < 4000}") // 真正的私有布尔值和字母； @Value("#{400 > 300 || 150 < 100}") // 真正的私有布尔值或； @Value("#{400 > 300 or 150 < 100}") // 真正的私有布尔值或字母； @Value("#{!true}") // false private boolean not; @Value("#{not true}") // false private boolean notAlphabetic; 与算术和关系运算符一样，所有逻辑运算符也有字母克隆。 ### 2.4。条件运算符 我们使用条件运算符根据某些条件注入不同的值：java @Value("#{2 > 1 ? 'a' : 'b'}") // "a" private String ternary ; 我们使用三元运算符在表达式中执行紧凑的 if-then-else 条件逻辑。在这个例子中，我们试图检查是否存在 true。三元运算符的另一个常见用途是检查某个变量是否为 null，然后返回变量值或默认值：java @Value("#{someBean.someProperty != null ? someBean.someProperty : ' default'}") 私有字符串三元； Elvis 运算符是一种缩短 Groovy 语言中使用的上述情况的三元运算符语法的方法。它也可用于 SpEL。这段代码等价于上面的代码： java @Value("#{someBean.someProperty ?: 'default'}") // 如果 someProperty 为空，将注入提供的字符串 private String elvis; ### 2.5。在 SpEL 中使用 Regex 我们可以使用 matches 运算符来检查字符串是否与给定的正则表达式匹配： ``java @Value("#{'100' matches '\d+' }") // true private boolean validNumericStringResult; @Value("#{'100fghdjf' 匹配 '\d+' }") // false private boolean invalidNumericStringResult; @Value("#{'valid alphabetic string' 匹配 '[a-zA-Z\s]+' }") // true private boolean validAlphabeticStringResult; @Value("#{'无效的字母字符串 #$1' 匹配 '[a-zA-Z\s]+' }") // false private boolean invalidAlphabeticStringResult; @Value("#{someBean.someValue 匹配 '\d+'}") // 如果 someValue 仅包含数字，则为 true private boolean validNumericValue; ### **2.6.访问 *List* 和 *Map* 对象** 在 SpEL 的帮助下，我们可以访问上下文中任何 *Map* 或 *List* 的内容。我们将创建新的 bean *workersHolder*，它将一些工人及其工资的信息存储在 *List* 和 *Map* 中：java @Component("workersHolder") public class WorkersHolder { private List 工人 = 新的链表<>(); private Map<String, Integer> SalaryByWorkers = new HashMap<>();公共 WorkersHolder() {workers.add("John"); worker.add("苏西"); worker.add("Alex"); worker.add("乔治"); SalaryByWorkers.put("约翰", 35000); SalaryByWorkers.put("Susie", 47000); SalaryByWorkers.put("Alex", 12000); SalaryByWorkers.put("乔治", 14000); } //Getter 和 setter } 现在我们可以使用 SpEL 访问集合的值：java @Value("#{workersHolder.salaryByWorkers['John']}") // 35000 private Integer johnSalary ; @Value("#{workersHolder.salaryByWorkers['George']}") // 14000 私有整数 georgeSalary; @Value("#{workersHolder.salaryByWorkers['Susie']}") // 47000 私有整数 susieSalary; @Value("#{workersHolder.workers[0]}") // John private String firstWorker; @Value("#{workersHolder.workers[3]}") // George private String lastWorker; @Value("#{workersHolder.workers.size()}") // 4 个私有整数 numberOfWorkers; **3。在 Spring 配置中使用** ---------------------------------- ### **3.1.引用 Bean** 在本例中，我们将了解如何在基于 XML 的配置中使用 SpEL。我们可以使用表达式来引用 bean 或 bean 字段/方法。例如，假设我们有以下类：java public class Engine { private int capacity;私人 int 马力；私人 int numberOfCylinders; // getter 和 setter } public class Car { private String make;私有 int 模型；私人引擎引擎；私人 int 马力； // Getter 和 setter } 现在我们创建一个应用程序上下文，其中表达式用于注入值：xml 看看 *someCar* bean。 *someCar* 的 *engine* 和 *horsePower* 字段使用的表达式分别是对 *engine* bean 和 *horsePower* 字段的 bean 引用。要对基于注释的配置执行相同操作，请使用 *@Value("#{expression}")*annotation。 ### **3.2。在配置中使用运算符** 本文第一部分中的每个运算符都可用于 XML 和基于注释的配置。但是，请记住，在基于 XML 的配置中，我们不能使用尖括号运算符“\<”。相反，我们应该使用字母别名，例如 *lt*（小于）或 *le*（小于或等于）。对于基于注解的配置，没有这样的限制：java public class SpelOperators { private boolean equal;私有布尔不等于；私有布尔大于等于或等于；私有布尔值和；私有布尔值或；私有字符串添加字符串； // getter 和 setter java @Override public String toString() { // toString 包含所有字段 } 现在我们将添加一个 *spelOperators* bean 到应用程序上下文：xml 从上下文中检索该 bean，然后我们可以验证值是否被正确注入：java ApplicationContext context = new ClassPathXmlApplicationContext("applicationContext.xml"); SpelOperators spelOperators = (SpelOperators) context.getBean("spelOperators"); 这里我们可以看到 *spelOperators* bean 的 *toString* 方法的输出：java [equal=true, notEqual=false, greaterThanOrEqual=true, and=true, or=true, addString=Some Some make] **4.以编程方式解析表达式** ------------------------------------------- 有时，我们可能想要在配置上下文之外解析表达式。幸运的是，使用 *SpelExpressionParser* 可以做到这一点。我们可以使用在前面的示例中看到的所有运算符，但应使用不带大括号和井号的运算符。也就是说，如果我们想在 Spring 配置中使用带有 *+* 运算符的表达式，则语法为 *#{1 + 1}* ；在配置之外使用时，语法就是 *1 + 1*。在以下示例中，我们将使用上一节中定义的 *Car* 和 *Engine* bean。 ### **4.1。使用 *ExpressionParser*** 让我们看一个简单的例子：java ExpressionParser expressionParser = new SpelExpressionParser();表达式表达式 = expressionParser.parseExpression("'任何字符串'");字符串结果 = (String) 表达式.getValue(); *ExpressionParser* 负责解析表达式字符串。在此示例中，SpEL 解析器将简单地将字符串 *'Any String'* 评估为表达式。不出所料，结果将是 *'Any String'*。与在配置中使用 SpEL 一样，我们可以使用它来调用方法、访问属性或调用构造函数：java Expression expression = expressionParser.parseExpression("'Any string'.length()");整数结果 = (Integer) 表达式.getValue(); 另外，我们可以调用构造函数，而不是直接对字面量进行操作：java Expression expression = expressionParser.parseExpression("new String('Any string').length()"); 我们也可以访问 *String* 类的 *bytes* 属性，以同样的方式，得到字符串的 byte\[\] 表示：java Expression expression = expressionParser.parseExpression("'任何字符串'.bytes"); byte[] 结果 = (byte[]) 表达式.getValue(); 我们可以链接方法调用，就像在普通 Java 代码中一样：java Expression expression = expressionParser.parseExpression("'Any string'.replace(" ", "").length() ");整数结果 = (Integer) 表达式.getValue(); 在这种情况下，结果将是 9，因为我们已经用空字符串替换了空格。如果我们不想转换表达式结果，我们可以使用通用方法 *T getValue(Class\<T\> desiredResultType)*，我们可以在其中提供我们想要返回的所需类的类型。请注意，如果返回的值不能强制转换为 *desiredResultType*，则会抛出 *EvaluationException*：java Integer result = expression.getValue(Integer.class); 最常见的用法是提供针对特定对象实例评估的表达式字符串：java Car car = new Car(); car.setMake("好制造商"); car.setModel("模型 3"); car.setYearOfProduction(2014); ExpressionParser expressionParser = new SpelExpressionParser();表达式表达式 = expressionParser.parseExpression("model");评估上下文上下文 = 新标准评估上下文（汽车）；字符串结果 = (String) 表达式.getValue(context); 在这种情况下，结果将等于 *car* 对象的 *model* 字段的值，“*Model 3*”。 *StandardEvaluationContext* 类指定表达式将针对哪个对象进行评估。上下文对象创建后无法更改。 *StandardEvaluationContext* 的构建成本很高，并且在重复使用期间，它会建立缓存状态，从而可以更快地执行后续表达式评估。由于缓存，如果根对象没有改变，最好尽可能重用 *StandardEvaluationContext*。但是，如果根对象反复更改，我们可以使用下面示例中所示的机制：java Expression expression = expressionParser.parseExpression("model");字符串结果 = (String) 表达式.getValue(car); 这里我们使用一个参数调用 *getValue* 方法，该参数表示我们想要应用 SpEL 表达式的对象。我们也可以像以前一样使用通用的 *getValue* 方法：java Expression expression = expressionParser.parseExpression("yearOfProduction > 2005"); boolean result = expression.getValue(car, Boolean.class); ### **4.2.使用 *ExpressionParser* 设置值** 在解析表达式返回的 *Expression* 对象上使用 *setValue* 方法，我们可以设置对象的值。 SpEL 将负责类型转换。默认情况下，SpEL 使用 *org.springframework.core.convert.ConversionService* 。我们可以在类型之间创建自己的自定义转换器。 *ConversionService* 是泛型感知的，因此我们可以将它与泛型一起使用。让我们看看我们在实践中是如何做到的：java Car car = new Car(); car.setMake("好制造商"); car.setModel("模型 3"); car.setYearOfProduction(2014);停车场 carPark = new CarPark(); carPark.getCars().add(汽车); StandardEvaluationContext context = new StandardEvaluationContext(carPark); ExpressionParser expressionParser = new SpelExpressionParser(); expressionParser.parseExpression("cars[0].model").setValue(context, "其他模型"); 生成的汽车对象将具有 *model* "*Other model* "，它是从 "*Model 3*" 更改的。 ### **4.3。解析器配置** 在下面的例子中，我们将使用这个类：java public class CarPark { private List cars = new ArrayList<>();
