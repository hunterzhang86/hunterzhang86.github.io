Spring Data Web 支持

 2023.1.8  2023.3.20  手册  4206  9 分钟  

1。概述 --------------- Spring MVC 和 Spring Data 在简化应用程序开发方面都做得很好自己的权利。但是，如果我们把它们放在一起呢？在本教程中，我们将看看 Spring Data 的 web 支持 和它的 resolvers 如何减少样板文件并使我们的控制器更具表现力。 在此过程中，我们将了解 Querydsl 以及它与 Spring Data 的集成是什么样的。 2. 背景介绍 ---------------------- Spring Data 的 web 支持是在标准 Spring MVC 之上实现的一组与 web 相关的特性平台，旨在为控制器层添加额外的功能。 Spring Data web 支持的功能是围绕几个 resolver 类构建的。解析器简化了与 Spring Data 存储库互操作的控制器方法的实现，并通过附加功能丰富了它们。这些功能包括从存储库层获取域对象，无需显式调用存储库实现，以及构造控制器响应，这些响应可以作为支持分页和排序。此外，对带有一个或多个请求参数的控制器方法的请求可以在内部解析为 Querydsl 查询。 3。一个演示 Spring Boot 项目 --------------------------------- 了解我们如何使用 Spring Data Web 支持为了改进控制器的功能，让我们创建一个基本的 Spring Boot 项目。我们的演示项目的 Maven 依赖是相当标准的，除了一些我们稍后会讨论的例外：xml <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter- data-jpa</artifactId> </dependency> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-web</artifactId> </dependency> <dependency> <groupId> com.h2database</groupId> <artifactId>h2</artifactId> <scope>runtime</scope> </dependency> <dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter -test</artifactId> <scope>test</scope> </dependency> 在这种情况下，我们包括了 [spring-boot-starter-web](https://search.maven.org/search? q=g:org.springframework.boot%20AND%20a:spring-boot-starter-web) ，因为我们将使用它来创建 RESTful 控制器，[spring-boot-starter-jpa](https:// /search.maven.org/search?q=g:org.springframework.boot%20AND%20a:spring-boot-starter-data-jpa) 用于实现持久层，以及*spring-boot-starter-test* 用于测试控制器API。由于我们将使用 H2 作为底层数据库，因此我们包含了 [com.h2database](https://search.maven.org/search?q=g:com .h2database%20AND%20a:h2) 也是如此。让我们记住，spring-boot-starter-web 默认启用 Spring Data web 支持。 因此，我们不需要创建任何额外的 @Configuration 类来让它在我们的应用程序中工作。相反，对于非 Spring Boot 项目，我们需要定义一个 @Configuration 类并使用 @EnableWebMvc 和 @EnableSpringDataWebSupport 注释对其进行注释。 ### 3.1。域类 现在，让我们在项目中添加一个简单的 UserJPA 实体类，这样我们就可以拥有一个可以使用的域模型： java @Entity 支持功能。由于该方法将 User 实例作为参数，我们最终可能会认为我们需要在请求中显式传递域对象。但是，我们没有。 Spring MVC 使用 DomainClassConverter 类将 id 路径变量转换为域类的 id 类型，并使用它从存储库层获取匹配的域对象。无需进一步查找。例如，对 http://localhost:8080/users/1 端点的 GET HTTP jsonPath("$.id").value("1")); } } 或者，我们可以使用 REST API 测试工具，例如 Postman 来测试方法。 DomainClassConverter 的好处是我们不需要在控制器方法中显式调用存储库实现。 通过简单地指定 id 路径变量以及可解析的域类实例，我们已经自动触发了域对象的查找。 5。 PageableHandlerMethodArgumentResolver 类 ------------------------------------------- ------------- Spring MVC 支持使用 [Pageable](https://docs.spring.io/spring-data/commons/docs/current/api/org/springframework /data/domain/Pageable.html) 类型的控制器和存储库。简单地说，Pageable 实例是一个保存分页信息的对象。因此，当我们将 Pageable 参数传递给控制器方法时，Spring MVC 使用 [PageableHandlerMethodArgumentResolver](https://docs.spring.io/spring-data/commons/docs/current/api/org/springframework/ data/web/PageableHandlerMethodArgumentResolver.html) 类将 Pageable 实例解析为 [PageRequest](https://docs.spring.io/spring-data/commons/docs/current/api/org/ springframework/data/domain/PageRequest.html) 对象， 这是一个简单的 Pageable 实现。 ### 5.1。使用 Pageable 作为控制器方法参数 要了解 PageableHandlerMethodArgumentResolver 类的工作原理，让我们向 UserController 类添加一个新方法：java @GetMapping("/users") public Page<User > findAllUsers(Pageable pageable) { return userRepository.findAll(pageable); } 与 findUserById() 方法相比，这里我们需要调用存储库实现来获取数据库中持久化的所有 User JPA 实体。由于该方法采用 Pageable 实例，因此它返回整个实体集的子集，存储在 Page<User> 对象中。 Page 对象是列表的子列表公开了几种方法的对象，我们可以使用这些方法来检索有关分页结果的信息，包括结果页面的总数，以及我们正在检索的页面的数量。默认情况下，Spring MVC 使用 PageableHandlerMethodArgumentResolver 类构造一个 PageRequest 对象，请求参数如下： * page : 我们要检索的页面的索引——该参数是零索引的，默认值为 0 * size ：我们要检索的页数 -- 默认值为 20 * sort ：我们可以使用以下格式对结果进行排序的一个或多个属性: property1,property2(,asc|desc) -- 例如 ?sort=name&sort=email,asc 例如对 [http://localhost:8080/user] 的 GET 请求(http://localhost:8080/users)s 端点将返回以下输出：plaintext { "content":[ { "id":1 , "name":"John" }, { "id":2, "name":"Robert" }, { "id":3, "name":"Nataly" }, { "id":4, " name":"Helen" }, { "id":5, "name":"Mary" }], "pageable":{ "sort":{ "sorted":false, "unsorted":true, "empty" :true }, "pageSize":5, "pageNumber":0, "offset":0, "unpage d":false, "paged":true }, "last":true, "totalElements":5, "totalPages":1, "numberOfElements":5, "first":true, "size":5, "number ":0, "sort":{ "sorted":false, "unsorted":true, "empty":true }, "empty":false } 我们可以看到，响应包括 first ， pageSize 、 totalElements 和 totalPages JSON 元素。这非常有用，因为前端可以使用这些元素轻松创建分页机制。此外，我们可以使用集成测试来检查 findAllUsers() 方法： java @Test public void whenGetRequestToUsersEndPoint_thenCorrectResponse() throws Exception { mockMvc.perform(MockMvcRequestBuilders.get("/users") .contentType( MediaType.APPLICATION_JSON_UTF8)) .andExpect(MockMvcResultMatchers.status().isOk()) .andExpect(MockMvcResultMatchers.jsonPath("$['pageable']['paged']").value("true")); } ### 5.2.自定义分页参数 在许多情况下，我们需要自定义分页参数。最简单的方法是使用 @PageableDefault 注释：java @GetMapping("/users") public Page<User> findAllUsers(@PageableDefault(value = 2, page = 0) Pageable pageable) { return userRepository.findAll(pageable); } 或者，我们可以使用 PageRequest 的 [of()](https://docs.spring.io/spring-data/commons/docs/current/api/org/springframework/data/domain /PageRequest.html#of-int-int-org.springframework.data.domain.Sort-) 静态工厂方法创建自定义 PageRequest 对象并将其传递给存储库方法：java @GetMapping( "/users") public Page<User> findAllUsers() { Pageable pageable = PageRequest.of(0, 5);返回 userRepository.findAll(pageable); } SortHandlerMethodArgumentResolver 类 ------------------------------------------- --------- 分页是有效管理大量数据库记录的事实上的方法。但是，如果我们不能以某种特定的方式对记录进行排序，就其本身而言，它是毫无用处的。为此，Spring MVC 提供了 SortHandlerMethodArgumentResolver 类。解析器自动从请求参数创建 Sort 实例或来自 @SortDefault 注释。 ### 6.1。使用 sort 控制器方法参数 要清楚了解 SortHandlerMethodArgumentResolver 类的工作原理，让我们将 findAllUsersSortedByName() 方法添加到控制器类：java
