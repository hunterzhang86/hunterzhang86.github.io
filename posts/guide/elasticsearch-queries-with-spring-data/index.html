使用 Spring Data 进行 Elasticsearch 查询

 2023.4.29  2023.5.11  手册  2195  5 分钟  

1。介绍

在 上一篇文章 中，我们演示了如何为项目配置和使用 Spring Data Elasticsearch。在本文中，我们将研究 Elasticsearch 提供的几种查询类型，还将讨论字段分析器及其对搜索结果的影响。

2。分析仪

默认情况下，所有存储的字符串字段都由分析器处理。一个分析器由一个分词器和几个分词过滤器组成，通常前面有一个或多个字符过滤器。

默认分析器通过常见的单词分隔符（例如空格或标点符号）拆分字符串，并将每个标记都放在小写中。它也忽略了常见的英语单词。

Elasticsearch 还可以配置为同时将字段视为已分析和未分析。

例如，在 Article 类中，假设我们将标题字段存储为标准分析字段。带有后缀 verbatim 的相同字段将存储为未分析字段：

1

@多字段（
  mainField = @Field(type = Text, fielddata = true),
  其他字段 = {
      @InnerField（后缀=“逐字”，类型=关键字）
  }
)
私有字符串标题；

在这里，我们应用 @MultiField 注释来告诉 Spring Data 我们希望该字段以多种方式被索引。主字段将使用名称 title 并将根据上述规则进行分析。

但是我们还提供了第二个注释 @InnerField ，它描述了 title 字段的附加索引。我们使用 FieldType.keyword 表示在执行字段的附加索引时我们不想使用分析器，并且应该使用带有后缀 verbatim 的嵌套字段来存储此值。

2.1。分析字段

让我们看一个例子。假设我们的索引中添加了一篇标题为“Spring Data Elasticsearch”的文章。默认分析器将在空格字符处分解字符串并生成小写标记：“spring”、“data”、 和“elasticsearch”。

现在我们可以使用这些术语的任意组合来匹配文档：

1
2
3

NativeSearchQuery searchQuery = new NativeSearchQueryBuilder()
  .withQuery(matchQuery("title", "elasticsearch data"))
  。建造（）;

2.2。非分析字段

未分析的字段不会被标记化，因此只能在使用 match 或 term 查询时作为一个整体进行匹配：

1
2
3

NativeSearchQuery searchQuery = new NativeSearchQueryBuilder()
  .withQuery(matchQuery("title.verbatim", "关于 Elasticsearch 的第二篇文章"))
  。建造（）;

使用匹配查询，我们只能按完整标题进行搜索，这也是区分大小写的。

3。匹配查询

匹配查询接受文本、数字和日期。

有三种类型的“匹配”查询：

  • 布尔值
  • 短语和
  • 短语前缀

在本节中，我们将探讨 boolean 匹配查询。

3.1。与布尔运算符匹配

boolean 是匹配查询的默认类型；您可以指定使用哪个布尔运算符（* 或* 是默认值）：

1
2
3
4
5

NativeSearchQuery searchQuery = new NativeSearchQueryBuilder()
  .withQuery(matchQuery("title","搜索引擎").operator(Operator.AND))
  。建造（）;
SearchHits<Article> 文章 = elasticsearchTemplate()
  .search(searchQuery, Article.class, IndexCoordinates.of("blog"));

通过使用 and 运算符指定标题中的两个术语，此查询将返回标题为“搜索引擎”的文章。但是，如果我们在只有一个条件匹配时使用默认 (or) 运算符进行搜索，会发生什么情况？

assertEquals(1,articles.getTotalHits());

“搜索引擎”文章仍然匹配，但它的得分较低，因为并非所有的词都匹配。

每个匹配词的得分总和就是每个结果文档的总得分。

在某些情况下，包含在查询中输入的稀有词条的文档将具有比包含多个常用词条的文档更高的排名。

3.2。模糊性

当用户在一个单词中打错字时，仍然可以通过指定 fuzziness 参数将其与搜索匹配，这允许不精确的匹配。

对于字符串字段，fuzziness 表示编辑距离：需要对一个字符串进行单个字符更改以使其与另一个字符串相同的次数。

1
2
3
4
5
6

NativeSearchQuery searchQuery = new NativeSearchQueryBuilder()
  .withQuery(matchQuery("title", "spring date elasticsearch")
  .operator（运算符.AND）
  .fuzziness（模糊性.ONE）
  .prefixLength(3))
  。建造（）;

prefix_length 参数用于提高性能。在这种情况下，我们要求前三个字符应该完全匹配，这减少了可能组合的数量。

5。短语搜索

相位搜索更为严格，尽管您可以使用 slop 参数对其进行控制。此参数告诉短语查询在仍将文档视为匹配项的同时允许相距多远的术语。

换句话说，它表示您需要移动术语以使查询和文档匹配的次数：

1
2
3

NativeSearchQuery searchQuery = new NativeSearchQueryBuilder()
  .withQuery(matchPhraseQuery("title", "spring elasticsearch").slop(1))
  。建造（）;

此处查询将匹配标题为“Spring Data Elasticsearch”的文档，因为我们将斜率设置为 1。

6。多匹配查询

当您想在多个字段中搜索时，您可以使用 QueryBuilders#multiMatchQuery() 在其中指定要匹配的所有字段：

1
2
3
4
5
6

NativeSearchQuery searchQuery = new NativeSearchQueryBuilder()
  .withQuery(multiMatchQuery("教程")
    .field("标题")
    .field("标签")
    .type(MultiMatchQueryBuilder.Type.BEST_FIELDS))
  。建造（）;

在这里，我们在 title 和 tags 字段中搜索匹配项。

请注意，这里我们使用“最佳领域”评分策略。它将字段中的最高分数作为文档分数。

8。概括

在本文中，我们讨论了已分析字段和未分析字段之间的区别，以及这种区别如何影响搜索。

我们还了解了 Elasticsearch 提供的几种查询类型，例如匹配查询、短语匹配查询、全文搜索查询和布尔查询。

Elasticsearch 提供了许多其他类型的查询，例如地理查询、脚本查询和复合查询。您可以在 Elasticsearch 文档 中阅读它们并探索 Spring Data Elasticsearch API 以使用这些代码中的查询。
