Joda-Time 简介

 2023.3.26  2023.4.19  手册  4198  9 分钟  
  1. 简介 ---------------- Joda-Time 是 Java 8 发布之前使用最广泛的日期和时间处理库。它的目的是提供一个直观的 API 来处理日期和时间，还解决了 Java 日期/时间 API 中存在的设计问题。 这个库中实现的中心概念在 Java 8 版本的发布中引入了 JDK 核心。 新的日期和时间 API 可以在 java.time 包中找到（[JSR-310](https ://jcp.org/en/jsr/detail?id=310))。这些特性的概述可以在这篇 文章 中找到。 Java 8 发布后，作者认为项目基本完成，建议尽可能使用 Java 8 API。 2. 为什么使用 Joda-Time？ --------------------- Java 8 之前的日期/时间 API 存在多个设计问题。问题之一是 Date 和 SimpleDateFormatter 类不是线程安全的。为了解决这个问题，Joda-Time 使用不可变类来处理日期和时间。 Date 类不代表实际日期，而是指定时间的瞬间，精度为毫秒。 Date 中的年份从 1900 年开始，而大多数日期操作通常使用从 1970 年 1 月 1 日开始的 Epoch 时间。此外，Date 的日、月和年偏移量是违反直觉的。天从 0 开始，而月从 1 开始。要访问它们中的任何一个，我们必须使用 Calendar 类。 Joda-Time 提供了一个干净流畅的 API 来处理日期和时间。 Joda-Time 还提供 支持 8 个日历系统，而 Java 仅提供 2 个：Gregorian -- java.util.GregorianCalendar和日语 -- java.util.JapaneseImperialCalendar。 3.设置--------要包含Joda-Time库的功能，我们需要从Maven Central添加以下依赖7Cga%7C1%7Cg%3A%22joda-time%22%20AND%20a%3A%22joda-time%22): xml <dependency> <groupId>joda-time</groupId> <artifactId>joda- time</artifactId> <version>2.10</version> </dependency> 4. 库概述 ------- Joda-Time 对 日期和时间的概念 使用 org.joda.time 包中的类。在这些类中最常用的是： * LocalDate -- 表示没有时间的日期 * LocalTime -- 表示没有时区的时间 * LocalDateTime -- 表示没有时区的日期和时间 * Instant -- 表示从 Java 纪元 1970-01-01T00:00:00Z 开始的精确时间点（以毫秒为单位） * Duration -- 表示两个时间点之间的持续时间（以毫秒为单位） * Period -- 类似到 Duration，但允许访问日期和时间对象的各个组件，如年、月、日等。 * Interval - 表示 2 个瞬间之间的时间间隔其他重要功能是 date 解析器和格式化程序。这些可以在 org.joda.time.format 包中找到。 日历系统和时区特定类可以在 org.joda.time.chrono 和 org.joda.time.tz 包中找到。让我们看一些示例，其中我们使用 Joda-Time 的主要功能来处理日期和时间。 5. 表示日期和时间 ------------------ ### 5.1.当前日期和时间当前日期，没有时间信息，可以通过 LocalDate类中的now()方法获得：java LocalDate currentDate = LocalDate.now (); 当我们只需要当前时间，没有日期信息时，我们可以使用 LocalTime 类：java LocalTime currentTime = LocalTime.now(); 要获得当前日期和时间的**表示而不考虑时区，我们可以使用LocalDateTime**：java LocalDateTime currentDateAndTime = LocalDateTime.now(); 现在，使用 currentDateAndTime，我们可以将其转换为其他类型的对象建模日期和时间。我们可以使用 toDateTime() 方法获取 DateTime 对象（考虑时区）。当不需要时间时，我们可以使用 toLocalDate() 方法将其转换为 LocalDate，而当我们只需要时间时，我们可以使用 toLocalTime() 来获取 LocalTime 对象：java DateTime dateTime = currentDateAndTime.toDateTime(); LocalDate localDate = currentDateAndTime.toLocalDate(); LocalTime localTime = currentDateAndTime.toLocalTime(); 以上所有方法都有一个重载方法，它接受一个 DateTimeZone 对象 来帮助我们表示指定时区中的日期或时间：java LocalDate currentDate = LocalDate.now(DateTimeZone. forID("美国/芝加哥")); 此外，Joda-Time 提供了与 Java 日期和时间 API 的出色集成。构造函数接受一个 java.util.Date 对象，而且我们可以使用 toDate() 方法返回一个 java.util.Date 对象：java LocalDateTime currentDateTimeFromJavaDate = new LocalDateTime(new Date ());日期 currentJavaDate = currentDateTimeFromJavaDate.toDate(); ### 5.2.自定义日期和时间 为了表示自定义日期和时间，Joda-Time 为我们提供了几个构造函数。我们可以指定以下对象： * 一个 Instant * 一个 Java Date 对象 * 一个使用 ISO 格式的日期和时间的 String 表示 * 日期和时间的一部分：年、月、日、小时、分、秒、毫秒java Date oneMinuteAgoDate = new Date(System.currentTimeMillis() - (60 * 1000));即时 oneMinutesAgoInstant = new Instant(oneMinuteAgoDate);日期时间 customDateTimeFromInstant = new DateTime(oneMinutesAgoInstant);日期时间 customDateTimeFromJavaDate = new DateTime(oneMinuteAgoDate);日期时间 customDateTimeFromString = new DateTime("2018-05-05T10:11:12.123");日期时间 customDateTimeFromParts = new DateTime(2018, 5, 5, 10, 11, 12, 123); 我们可以定义自定义日期和时间的另一种方法是解析 ISO 格式的日期和时间的给定 String 表示：java DateTime parsedDateTime = DateTime.parse("2018-05-05T10 :11:12.123"); 我们还可以通过定义自定义 DateTimeFormatter 来解析日期和时间的自定义表示： java DateTimeFormatter dateTimeFormatter = DateTimeFormat.forPattern("MM/dd/yyyy HH:mm:ss"); DateTime parsedDateTimeUsingFormatter = DateTime.parse("05/05/2018 10:11:12", dateTimeFormatter); 6. 使用日期和时间 ----------------------------------------- ### 6.1.使用 Instant Instant 表示从 1970-01-01T00:00:00Z 到给定时刻的毫秒数。例如，可以使用默认构造函数或方法 now() 获取当前时刻：java Instant instant = new Instant(); Instant.now(); 要为自定义时刻创建 Instant，我们可以使用任一构造函数或使用方法 ofEpochMilli() 和 ofEpochSecond()：java Instant InstantFromEpochMilli = Instant.ofEpochMilli (milliesFromEpochTime); Instant InstantFromEpocSeconds = Instant.ofEpochSecond(secondsFromEpochTime); 构造函数接受一个 String 表示 ISO 格式的日期和时间，一个 Java Date 或一个 long 值表示从 1970-01-01T00:00:00Z 开始的毫秒数：java Instant InstantFromString = new Instant("2018-05-05T10:11:12"); Instant InstantFromDate = new Instant(oneMinuteAgoDate); Instant InstantFromTimestamp = new Instant(System.currentTimeMillis() - (60 * 1000)); 当日期和时间表示为 String 时，我们可以选择使用所需格式解析 String：java Instant parsedInstant = Instant.parse("05/05/2018 10:11 :12", dateTimeFormatter); 现在我们知道 Instant 代表什么以及如何创建它，让我们看看如何使用它。为了与 Instant 对象进行比较，我们可以使用 compareTo()，因为它实现了 Comparable 接口，但我们也可以使用 Instant 也实现的 ReadableInstant 接口中提供的 Joda-Time API 方法：` ``java assertTrue(instantNow.compareTo(oneMinuteAgoInstant) > 0); assertTrue(instantNow.isAfter(oneMinuteAgoInstant)); assertTrue(oneMinuteAgoInstant.isBefore(instantNow)); assertTrue(oneMinuteAgoInstant.isBeforeNow()); assertFalse(oneMinuteAgoInstant.isEqual(instantNow)); 另一个有用的特性是 ***Instant* 可以转换为 *DateTime* 对象或 Java *Date*** 事件：java DateTime dateTimeFromInstant = instant.toDateTime();日期 javaDateFromInstant = instant.toDate(); 当我们需要访问日期和时间的一部分时，比如年、小时等，我们可以使用 *get()* 方法并指定 *DateTimeField*:java int year =即时.get(DateTimeFieldType.year()); int 月 = Instant.get(DateTimeFieldType.monthOfYear()); int day = instant.get(DateTimeFieldType.dayOfMonth()); int hour = instant.get(DateTimeFieldType.hourOfDay()); 现在我们已经介绍了 *Instant* 类，让我们看看如何使用 *Duration* 、 *Period* 和 *Interval* 的一些示例。 ### 6.2。使用 *Duration* 、 *Period* 和 *Interval* 一个 *Duration* 表示两个时间点之间的时间（以毫秒为单位），或者在这种情况下它可以是两个 *Instants* 。 **当我们需要在不考虑时间顺序和时区的情况下向另一个 *Instant* 添加或减去特定时间量时，我们将使用它**：java long currentTimestamp = System.currentTimeMillis(); long oneHourAgo = currentTimestamp - 24601000; Duration 持续时间 = new Duration(oneHourAgo, currentTimestamp); Instant.now().plus(持续时间); 另外，我们可以确定持续时间代表多少天、小时、分钟、秒或毫秒：java long durationInDays = duration.getStandardDays(); long durationInHours = duration.getStandardHours(); long durationInMinutes = duration.getStandardMinutes(); long durationInSeconds = duration.getStandardSeconds(); long durationInMilli = duration.getMillis(); *Period* 和 *Duration* 之间的主要区别在于 ***Period* 是根据其日期和时间组成部分（年、月、小时等）定义的，并不代表确切的毫秒**。使用 *Period* 日期和时间计算时，**将考虑时区和夏令时**。例如，将 1 个月的 *Period* 添加到 2 月 1 日将导致日期表示为 3 月 1 日。通过使用 *Period*，图书馆将考虑闰年。如果我们要使用 *Duration*，我们的结果将不正确，因为 *Duration* 表示不考虑时间顺序或时区的固定时间量：java Period period =新的 Period().withMonths(1); LocalDateTime datePlusPeriod = localDateTime.plus(period); 一个 *Interval* ，顾名思义，表示由两个 *Instant* 对象表示的两个固定时间点之间的日期和时间间隔：java Interval interval = new Interval(oneMinuteAgoInstant, instantNow); 当我们需要检查两个区间是否重叠或计算它们之间的差距时，该类很有用。 *overlap()* 方法将返回重叠的 *Interval* 或 *null* 当它们不重叠时：java Instant startInterval1 = new Instant("2018-05-05T09:00:00.000");即时 endInterval1 = new Instant("2018-05-05T11:00:00.000");间隔 interval1 = new Interval(startInterval1, endInterval1);即时 startInterval2 = new Instant("2018-05-05T10:00:00.000");即时 endInterval2 = new Instant("2018-05-05T11:00:00.000");间隔 interval2 = new Interval(startInterval2, endInterval2);间隔重叠Interval = interval1.overlap(interval2); 可以使用 *gap()* 方法计算间隔之间的差异，当我们想知道一个间隔的结束是否等于另一个间隔的开始时，我们可以使用 *abuts()* 方法：java assertTrue(interval1.abuts(new Interval(new Instant("2018-05-05T11:00:00.000"), new Instant("2018-05-05T13:00:00.000")))); ### 6.3.日期和时间操作 一些最常见的操作是日期和时间的加、减和转换。该库为每个类 *LocalDate* 、 *LocalTime* 、 *LocalDateTime* 和 *DateTime* 提供了特定的方法。重要的是要注意这些类是不可变的，因此每个方法调用都会创建一个其类型的新对象。让我们以 *LocalDateTime* 为当前时刻并尝试更改其值：java LocalDateTime currentLocalDateTime = LocalDateTime.now(); 为了给 *currentLocalDateTime* 添加额外的一天，我们使用 *plusDays()* 方法：java LocalDateTime nextDayDateTime = currentLocalDateTime.plusDays(1); 我们也可以使用 *plus()* 方法将 *Period* 或 *Duration* 添加到 *currentLocalDateTime:*java Period oneMonth = new Period().withMonths(1); LocalDateTime nextMonthDateTime = currentLocalDateTime.plus(oneMonth); 其他日期和时间组件的方法类似，例如，*plusYears()* 用于添加额外的年份，plusSeconds() 用于添加更多的秒数等等。要从我们的 *currentLocalDateTime* 中减去一天，我们可以使用 *minusDays()* 方法：java LocalDateTime previousDayLocalDateTime = currentLocalDateTime.minusDays(1); 除了用日期和时间进行计算外，我们还可以设置日期或时间的各个部分。例如，可以使用 *withHourOfDay()* 方法将小时设置为 10。以前缀 *"with"* 开头的其他方法可用于设置该日期或时间的组件：java LocalDateTime currentDateAtHour10 = currentLocalDateTime .withHourOfDay(0) .withMinuteOfHour(0) .withSecondOfMinute(0) .withMillisOfSecond (0); 另一个重要方面是我们可以将日期和时间类类型转换为另一种。为此，我们可以使用库提供的特定方法： * *toDateTime()* -- 将 *LocalDateTime* 转换为 *DateTime* 对象 * *toLocalDate()* -- 将 *LocalDateTime* 转换为 *LocalDate* 对象* *toLocalTime() -- 将 LocalDateTime 转换为 LocalTime 对象* * *toDate()* -- 将 *LocalDateTime* 转换为 Java *Date* 对象 7. 使用时区 ----------- --------------- Joda-Time 使我们可以轻松地使用不同的时区并在它们之间进行更改。我们有 *DateTimeZone* 抽象类，用于表示有关时区的所有方面。 **Joda-Time 使用的默认时区是从 *user.timezone* Java 系统属性中选择的。** 库 API 允许我们为每个类或计算单独指定应使用的时区。例如，我们可以创建一个 LocalDateTime 对象当我们知道我们将在整个应用程序中使用特定的时区时，我们可以设置默认时区：java DateTimeZone.setDefault(DateTimeZone.UTC); 从现在开始，所有的日期和时间操作，如果没有另外指定，都将在 UTC 时区中表示。要查看所有可用时区，我们可以使用方法 *getAvailableIDs():*java DateTimeZone.getAvailableIDs() 当我们需要表示特定时区的日期或时间时，我们可以使用任何类 *LocalTime* ， *LocalDate* ， *LocalDateTime* ， *DateTime* 并在构造函数中指定 *DateTimeZone* 对象：java DateTime dateTimeInChicago = new DateTime(DateTimeZone.forID("America/Chicago")) ; DateTime dateTimeInBucharest = new DateTime(DateTimeZone.forID("Europe/Bucharest")); LocalDateTime localDateTimeInChicago = new LocalDateTime(DateTimeZone.forID("America/Chicago")); 另外，在这些类之间进行转换时，我们可以指定所需的时区。方法 *toDateTime()* 接受 *DateTimeZone* 对象和 *toDate()* 接受 java.util.TimeZone 对象： ;日期转换日期 = localDateTimeInChicago.toDate(TimeZone.getTimeZone("Europe/Bucharest")); 8. 结论 ------------- Joda-Time 是一个很棒的库，其主要目标是解决 JDK 中有关日期和时间操作的问题。它很快成为日期和时间处理的 de facto 库，最近它的主要概念在 Java 8 中被引入。重要的是要注意，作者认为它“是一个基本完成的项目” 并建议迁移现有代码以使用 Java 8 实现。** 文章的源代码可用 [在 GitHub 上] （https://github.com/hunterzhang86/tutorials/tree/master/core-java-modules/core-java-date-operations-1）。 Java 底层 ### 通过 Learn Spring 课程开始使用 Spring 5 和 Spring Boot 2： >> 查看课程** 通用页脚横幅！使用 SPRING 构建您的 API - 书籍封面 ！使用 SPRING 构建您的 API - 图标 学习使用 Spring 构建 API **？[下载电子书](/休息API弹簧指南）
