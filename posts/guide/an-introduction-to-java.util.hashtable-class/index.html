java.util.Hashtable 类简介

 2023.3.1  2023.5.11  手册  4042  9 分钟  

1。概述

Hashtable 是最古老的哈希表实现Java 中的数据结构。 HashMap 是第二种实现，它是在 JDK 1.2 中引入的。

这两个类都提供了类似的功能，但也有一些小的差异，我们将在本教程中进行探讨。

2。何时使用 哈希表

假设我们有一本字典，其中每个单词都有其定义。此外，我们需要快速从字典中获取、插入和删除单词。

因此， Hashtable （或 HashMap ）是有意义的。单词将是 Hashtable 中的键，因为它们应该是唯一的。另一方面，定义将是价值观。

4。 hashCode() 的重要性

要用作 Hashtable 中的键，对象不得违反 hashCode() 合同。 简而言之，相等的对象必须返回相同的代码。为了理解为什么让我们看一下哈希表是如何组织的。

Hashtable 使用数组。数组中的每个位置都是一个“桶”，可以为空或包含一个或多个键值对。计算每对的索引。

但是为什么不按顺序存储元素，在数组末尾添加新元素呢？

关键是按索引查找元素比通过顺序比较遍历元素要快得多。因此，我们需要一个将键映射到索引的函数。

4.1。直接地址表

这种映射的最简单示例是直接地址表。这里的键用作索引：

指数（k）=k，
其中 k 是一个键

键是唯一的，即每个桶包含一个键值对。当整数键的可能范围相当小时，此技术适用于整数键。

但是我们这里有两个问题：

  • 首先，我们的键不是整数，而是 Word 对象
  • 其次，如果它们是整数，没有人会保证它们很小。假设键是 1、2 和 1000000。我们将有一个大小为 1000000 的大数组，只有三个元素，其余的将是浪费的空间

hashCode() 方法解决了第一个问题。

Hashtable 中的数据操作逻辑解决了第二个问题。

让我们深入讨论一下。

4.2。 hashCode() 方法

任何 Java 对象都继承了 hashCode() 方法，该方法返回一个 int 值。该值是根据对象的内部内存地址计算得出的。默认情况下 hashCode() 返回不同对象的不同整数。

因此，任何键对象都可以使用 hashCode() 转换为整数。但是这个整数可能很大。

4.3。缩小范围

get() 、 put() 和 remove() 方法包含解决第二个问题的代码——减少可能整数的范围。

该公式计算键的索引：

1

int index = (hash & 0x7FFFFFFF) % tab.length;

其中tab.length 是数组大小，hash 是键的hashCode() 方法返回的数字。

正如我们所见，index 提醒了 hash 除以数组大小。请注意，相同的哈希码产生相同的索引。

4.4。碰撞

此外，即使不同的哈希码也可以产生相同的索引。我们将此称为碰撞。为了解决冲突，Hashtable 存储键值对的LinkedList。

这种数据结构称为链式哈希表。

4.5。负载系数

很容易猜测碰撞会减慢元素的操作。要获得一个条目，仅仅知道它的索引是不够的，但我们需要遍历列表并与每个条目进行比较。

因此，减少碰撞次数很重要。数组越大，碰撞的机会就越小。 负载因子决定了数组大小和性能之间的平衡。 默认情况下，它是 0.75，这意味着当 75% 的存储桶不为空时，数组大小会翻倍。该操作由 rehash() 方法执行。

但是，让我们回到键。

另外，请注意，我们不关心 String 、 Integer 、 Long 或其他包装器类型的键。 equal() 和 hashCode() 方法都已经在包装类中被覆盖。

6。 Hashtable 与 HashMap

Hashtable 和 HashMap 提供非常相似的功能。

它们都提供：

*快速失败的迭代

  • 不可预测的迭代顺序

但也有一些区别：

  • HashMap 不提供任何 *Enumeration，而 * Hashtable 不提供快速失败 Enumeration
  • Hashtable 不允许 null 键和 null 值，而 HashMap 确实允许一个 null 键和任意数量的 null 值
  • Hashtable 的方法是同步的，而 HashMaps 的方法是不同步的
